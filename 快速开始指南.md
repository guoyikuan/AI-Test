# AI驱动的自动化测试框架 - 快速开始指南

## 🚀 安装和配置

### 1. 安装依赖

```bash
# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装Python依赖
pip install -r requirements.txt

# 安装Playwright浏览器
playwright install chromium
```

### 2. 配置环境变量

```bash
# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件，填入以下信息：
# - OPENAI_API_KEY 或 ANTHROPIC_API_KEY
# - Power BI认证信息（可选）
# - 测试环境URL
```

### 3. 验证安装

```bash
# 检查Playwright
playwright --version

# 检查Python包
pytest --version
```

## 📝 使用示例

### 1. 生成测试用例

```python
from llm_services.test_generation.test_generator import TestCaseGenerator

# 创建生成器
generator = TestCaseGenerator()

# 从需求生成测试用例
result = generator.generate_from_requirement(
    requirement="""
    用户登录功能：
    1. 输入用户名和密码
    2. 点击登录按钮
    3. 验证登录成功
    """,
    feature_name="用户登录",
    output_format="pytest"
)

print(f"测试用例已生成: {result['file']}")
```

### 2. 运行UI测试

```bash
# 运行所有UI测试
pytest tests/ui/ -v

# 运行特定测试
pytest tests/ui/test_example.py::test_login_success -v

# 使用自愈机制
pytest tests/ui/ --self-healing
```

### 3. 使用自愈定位器

```python
from playwright.sync_api import sync_playwright
from core.self_healing.self_healing import SelfHealingLocator
from core.llm.llm_client import LLMClient

with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page()
    
    # 创建自愈定位器
    healing = SelfHealingLocator(page, LLMClient())
    
    # 使用自愈定位器（如果原始定位器失败，会自动尝试新定位器）
    element = healing.find_element(
        "input[name='username']",
        "用户名输入框"
    )
    
    if element:
        element.fill("testuser")
```

### 4. 缺陷分析

```python
from llm_services.defect_analysis.defect_analyzer import DefectAnalyzer

analyzer = DefectAnalyzer()

# 分析测试失败
analysis = analyzer.analyze_failure(
    test_name="test_login",
    error_message="Element not found: input[name='username']",
    stack_trace="...",
    screenshot_path="screenshots/failure.png",
    test_steps=["导航到登录页", "查找用户名输入框"]
)

print("可能原因:", analysis["possible_causes"])
print("验证步骤:", analysis["verification_steps"])
print("修复建议:", analysis["fix_suggestions"])
```

### 5. 提示词微调

```python
from llm_services.prompt_tuning.prompt_tuner import PromptTuner, FeedbackCollector

# 创建调优器
tuner = PromptTuner()

# 收集反馈
collector = FeedbackCollector()
feedback = collector.collect_feedback(
    prompt_type="defect_analysis",
    prompt_version="1.0.0",
    llm_response="分析结果...",
    actual_result="实际结果...",
    user_rating=4,
    user_comment="分析准确，但建议更详细"
)

# 优化提示词
optimized = tuner.optimize_prompt(
    prompt_type="defect_analysis",
    feedback={
        "issues": ["输出格式不够清晰"],
        "suggestions": ["添加结构化输出"],
        "success_rate": 0.85,
        "avg_response_time": 2.5
    }
)

print(f"新版本: {optimized['version']}")
```

## 🔄 CI/CD集成

### GitHub Actions

1. 将工作流文件复制到 `.github/workflows/`:

```bash
mkdir -p .github/workflows
cp ci_cd/github_actions/test_workflow.yml .github/workflows/test.yml
```

2. 在GitHub仓库设置Secrets:
   - `OPENAI_API_KEY`
   - `ANTHROPIC_API_KEY`
   - `POWER_BI_CLIENT_ID`
   - `POWER_BI_CLIENT_SECRET`
   - `POWER_BI_TENANT_ID`

3. 推送代码，GitHub Actions会自动运行测试

## 🎯 最佳实践

### 1. 测试用例编写

- 使用Page Object Model模式
- 利用自愈定位器提高稳定性
- 添加详细的测试步骤和断言

### 2. LLM使用

- 合理设置temperature参数
- 提供清晰的上下文信息
- 定期优化提示词

### 3. 自愈机制

- 优先使用稳定的定位器（role, testid）
- 为元素添加描述信息
- 监控自愈成功率

### 4. 缺陷分析

- 提供完整的错误信息
- 包含截图和网络日志
- 记录测试步骤

## 🐛 故障排查

### 常见问题

1. **Playwright浏览器未安装**
   ```bash
   playwright install
   ```

2. **LLM API调用失败**
   - 检查API密钥是否正确
   - 检查网络连接
   - 查看API配额

3. **Power BI推送失败**
   - 验证认证信息
   - 检查数据集ID

4. **自愈机制不工作**
   - 确认LLM配置正确
   - 检查DOM结构是否可访问
