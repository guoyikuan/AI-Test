# AI驱动的自动化测试框架 - 项目架构说明

## 🏗️ 架构概览

本框架整合了 **Playwright + Pytest + LLM + Robot Framework + LangChain + Power BI Dashboard**，实现了智能化的自动化测试能力。

## 🎯 核心功能模块

### 1. 统一UI自动化基座

**文件**: `core/base/page_base.py`

- **PageBase类**: 页面对象模型基类
  - 智能等待策略（`wait_for_element`）
  - 通用操作封装（`click`, `fill`, `get_text`）
  - 页面快照（用于LLM分析）
  
- **SmartWaitStrategy类**: 智能等待策略
  - 条件等待
  - 自适应超时

### 2. LLM集成能力

**文件**: `core/llm/llm_client.py`

- **LLMClient类**: 统一LLM客户端
  - 支持OpenAI和Anthropic
  - 文本生成
  - 上下文分析
  - 缺陷分析
  - 定位推荐
  - 用例生成

### 3. 智能定位与自愈

**文件**: `core/self_healing/self_healing.py`

- **SelfHealingLocator类**: 自愈定位器
  - 定位失败时自动分析
  - 使用LLM推荐新定位策略
  - 自动尝试新定位器
  - 记录自愈历史

- **LocatorStrategy类**: 定位策略推荐
  - 优先级：role > testid > label > placeholder > text > css/xpath

### 4. LLM驱动用例生成

**文件**: `llm_services/test_generation/test_generator.py`

- **TestCaseGenerator类**: 测试用例生成器
  - 从需求文档生成用例
  - 从用户故事生成用例
  - 批量生成
  - 支持Pytest和Robot Framework格式

- **RequirementParser类**: 需求解析器
  - 解析Markdown格式需求

### 5. 缺陷根因分析

**文件**: `llm_services/defect_analysis/defect_analyzer.py`

- **DefectAnalyzer类**: 缺陷分析器
  - 分析测试失败
  - 提取可能原因
  - 提供验证步骤
  - 生成修复建议
  - 保存分析报告

- **FailurePatternDetector类**: 失败模式检测
  - 检测常见失败模式（timeout, network, assertion等）

### 6. Power BI数据收集

**文件**: `dashboard/power_bi_collector.py`

- **PowerBICollector类**: 数据收集器
  - 收集测试结果
  - 收集LLM指标
  - 收集自愈指标
  - 收集提示词调优数据
  - 批量推送数据

### 7. 持续集成

**文件**: `ci_cd/github_actions/test_workflow.yml`

- GitHub Actions工作流
  - 多浏览器并行测试
  - 多Python版本测试
  - 自动生成LLM测试用例
  - 失败时自动分析
  - 上传测试结果和报告

## 🔄 工作流程

### 1. 用例生成流程

```
需求文档/用户故事 
  → RequirementParser解析
  → LLMClient生成测试用例
  → TestCaseGenerator保存
  → tests/generated/
```

### 2. 测试执行流程

```
测试用例执行
  → Playwright运行
  → 定位失败？
    → SelfHealingLocator自愈
    → LLM推荐新定位器
    → 重试
  → 测试失败？
    → DefectAnalyzer分析
    → LLM根因分析
    → 生成分析报告
  → PowerBICollector收集数据
  → 推送到Power BI
```

### 3. 反馈闭环流程

```
测试执行结果
  → Power BI Dashboard
  → 分析LLM建议质量
  → 分析自愈成功率
  → 优化提示词
  → 更新LLM服务
  → 持续改进
```

## 🚀 使用示例

### 1. 生成测试用例

```python
from llm_services.test_generation.test_generator import TestCaseGenerator

generator = TestCaseGenerator()
result = generator.generate_from_requirement(
    requirement="用户登录功能，需要验证用户名和密码",
    feature_name="用户登录",
    output_format="pytest"
)
```

### 2. 使用自愈定位器

```python
from core.self_healing.self_healing import SelfHealingLocator
from core.llm.llm_client import LLMClient

healing = SelfHealingLocator(page, LLMClient())
element = healing.find_element(
    "input[name='username']",
    "用户名输入框"
)
```

### 3. 缺陷分析

```python
from llm_services.defect_analysis.defect_analyzer import DefectAnalyzer

analyzer = DefectAnalyzer()
analysis = analyzer.analyze_failure(
    test_name="test_login",
    error_message="Element not found",
    stack_trace="...",
    screenshot_path="screenshots/failure.png"
)
```

   - Token使用量
   - 成本

3. **自愈指标**
   - 自愈触发次数
   - 自愈成功率
   - 自愈耗时

4. **质量指标**
   - LLM分析准确度
   - 提示词优化效果
   - 用例生成质量
